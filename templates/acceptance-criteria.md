# Acceptance Criteria

## Behavior

- [ ] Given <context>, when <action>, then <observable result>.
- [ ] Given <context>, when <action>, then <observable result>.

## Compatibility

- [ ] Existing behavior remains unchanged for <case>.
- [ ] Public contract remains compatible or breaking change is documented and approved.

## Validation

- [ ] Automated validation covers <behavior>.
- [ ] Manual validation covers <behavior> when automation is not feasible.

## Non-Functional Criteria

- [ ] Security risk reviewed when applicable.
- [ ] Performance risk reviewed when applicable.
- [ ] Documentation updated when user, operator, or maintainer impact exists.

## Done Conditions

- [ ] Commands and results recorded.
- [ ] Risks remaining recorded.
- [ ] Review completed.
